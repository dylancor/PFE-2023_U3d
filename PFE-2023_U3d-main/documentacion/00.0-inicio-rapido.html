Requerimientos, herramientas y librerías de la unidad (Jul-23)

  • (entorno) NodeJs 18.16.1 LTS (soporte extendido)
  • (gestor de paquetes) Npm 8.3.1 ó Yarn 1.22.19
  • (plantilla de nueva app) Vite 4.3.9
  • (librería cliente http) Axios 1.4.0
  • (librería react de bootstrap) Reactstrap 9.2.0 (opcional)
Diferencias entre versiones

Pueden existir diferencias menores de versiones entre las presentadas. Considere que sean versiones con numeración cercana y actualizada, nunca inferiores a los requisitos.

Instalación base

Se requiere NodeJS como base para todo lo que se trabajará, puede encontrarse en la página principal de nodejs

Instalación de Yarn

De preferencia, se utilizará el gestor de paquetes Yarn.

corepack enable
npm i -g corepack

Creación de proyecto React con plantilla de aplicación Vite a través del gestor de paquetes Yarn

A través del CLI, debe posicionarse en el directorio donde desea que se cree el nuevo proyecto para crear la plantilla de aplicación Vite.

yarn create vite mi-nuevo-proyecto --template react
Se puede utilizar npm con sus comandos equivalentes; para ello, verificar la documentación de instalación de cada tecnología

Reconstrución de paquetes en proyecto nuevo

Una vez creado el nuevo proyecto, deben descargarse las librerías necesarias que quedarán almacenadas en el directorio node_modules.

Las librerías, configuraciones adicionales, nombres y parámetros del proyecto para ejecutar o compilar se encuentran definidas en el archivo package.json.

Para continuar, es recomendable continuar con las indicaciones que Vite entrega por consola al finalizar la creación del proyecto.

cd mi-nuevo-proyecto
yarn

Ejecutando servidor web con Vite + React

En el directorio del proyecto (mi-nuevo-proyecto) se ejecutará el servidor en entorno de desarrollo a través del siguiente comando:

yarn dev

Resultado aproximado:

yarn run v1.22.19
$ vite

  VITE v4.3.9  ready in 608 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h to show help
Acceder a la aplicación web

La url presentada en Local será el servidor web por donde se accede al sitio web / aplicación de React. En este caso, se debe ingresar desde un navegador web a http://localhost:5173

Proyecto en producción

Para generar un proyecto compilado para producción, se utiliza el comando yarn build, el resultado de los assets y las versiones "minimalistas" de html, css y js se copiarán en el directorio dist/ del proyecto.

Los archivos resultantes pueden ser usados en cualquier servidor web (apache, nginx, iis, etc.)

¿Cómo detener el servidor en React?

Al igual que casi cualquier app en CLI, se utiliza la combinación de teclas Ctrl+C

Estructura del proyecto

La estructura del nuevo proyecto debería ser aproximadamente la siguiente:

. 
├── dist/ 
├── node_modules/ 
├── public/ 
│   └── vite.svg
├── src/ 
│   ├── assets/ 
│   │   └── react.svg 
│   ├── App.css 
│   ├── App.jsx 
│   ├── index.css 
│   └── main.jsx
├── .eslintrc.cjs 
├── .gitignore 
├── index.html 
├── package.json 
├── vite.config.js
└── yarn.lock

Cabe señalar la funcionalidad de cada directorio y/o archivo:

  • dist/: Directorio generado cuando se genera un compilado para producción, puede que no sea mostrado si nunca se ha ejecutado yarn build

  • node_modules/: Directorio donde se almacenan las librerías necesarias para el funcionamiento del servidor web y la aplicación de React. No se debe modificar ni tampoco enviar para la evaluación.

  • public/: Directorio donde se almacenan elementos que deseamos importar desde la página web desde su index.html, aquí se incluyen archivos css externos como íconos a través de hojas de estilo, algunas imágenes como el ícono del sitio y webfonts.

  • src/: Directorio donde se trabajará todo el código fuente relativo a la aplicación de React por sí. En este directorio existen archivos importantes, tales como:

    • main.jsx: Punto de entrada de la aplicación, es el primer elemento de Javascript que será cargado y que posteriormente invocará a la "App" (componente)
    • App.jsx: Componente principal donde se muestra la página web. Es el punto de partida para el desarrollador.

  • .eslintrc.cjs: Archivo de reglas para la escritura estricta de Javascript.

  • .gitignore: Archivo de configuración git que permite indicar archivos y/o directorios que se omitirán al subir el repositorio a un servidor remoto. Se suele recomendar a los alumnos añadir al comienzo las siguientes líneas (si es que utilizan Visual Studio Code o Webstorm) para omitir directorios creados por editores o IDE's.

.vscode/
.idea/

  • index.html: Documento de hipertexto que se cargará al inicio de la aplicación, donde se encuentra la etiqueta de origen que cargará los componentes de React.

  • package.json: Documento propio de proyectos de node que indica configuraciones, librerías, dependencias y compiladores de aplicaciones javascript.

  • vite.config.js: Configuración para plantilla inicial vite.

Añadiendo librerías (requeridas y otras) al proyecto

Para añadir librerías, se utiliza la sintaxis desde el CLI del gestor de paquetes seleccionado, dentro del directorio del proyecto.

Añadir librería

Añadir localmente (sólo proyecto)

yarn add <nombre_libreria>

Añadir globalmente (librería disponible para todos los proyectos en el equipo)

yarn global add <nombre_libreria>

Añadir librerías requeridas del proyecto

Se añadirán al proyecto las librerías axios y reactstrap desde el repositorio de Yarn.

Nota

El servidor web debe encontrarse detenido para añadir nuevas librerías sin problemas

yarn add axios
